# AI Coding Agent 框架 — 领域模型

## 核心概念

### Agent（代理）
AI Coding Agent 是基于 **Query Loop + Tool Use** 的任务执行实体。

- **主 Agent（Orchestrator）**：中心化规划、审批与质量控制的决策者。制定任务计划，通过 Tool Call 接口调度子 Agent 执行。
- **子 Agent（Worker）**：以 Tool Call 方式受控执行的工作单元。
  - 拥有**全量 Tool 访问权限**
  - **生命周期**：一次调用用完即销毁，不维护跨步骤状态
  - **记忆权限**：只读（只能查询记忆系统，不能写入）
  - **执行模式**：主 Agent 定计划 → 子 Agent 自己规划执行细节

### Skill（技能）
多个 **原子 Tool** 的**编排序列**，是框架的任务执行的基本能力单元。

- 解决：Tool 数量增长后的检索噪声与功能重叠
- 元信息：意图标签、适用边界、示例
- 注意：Skill 是编排（orchestration），不是 Tool 的 wrapper

### Tool（工具）
不可再分的最小能力单元。原子操作。

### Skill 目录（Skill Catalog）
存放所有 Skill 的注册中心。

## Skill 分层路由体系

```
Tool (原子操作) → Skill (多个 Tool 编排) → Skill 目录 (注册中心)
```

### 两阶段召回机制

| 阶段 | 方法 | 产出 |
|------|------|------|
| Stage 1 召回 | 意图分类 + 元信息标签匹配 | 候选 Skill 集合 |
| Stage 2 精排 | 打分函数（非 LLM） | 最优 Skill |

设计目标：解决 Skill 自进化增长下的检索噪声、功能重叠与 Token 成本过高问题。

## 记忆系统（自进化记忆沉淀）

### 记忆类型

| 类型 | 内容 | 例子 |
|------|------|------|
| **事实记忆** | API 参数、项目配置等静态知识 | `Tool X 的参数格式` |
| **程序性记忆** | 步骤性 How-to 经验 | `处理 Y 问题的标准步骤` |
| **情景记忆** | 特定情境下的执行经验 | `上次处理 Z 时踩了坑` |

### 生命周期闭环

```
执行 → 反思 → 提炼 → 分类存储 → 索引更新 → 按需复用
```

### 反思触发策略（混合模式）

- **成功时**：提炼程序性经验（正样本）
- **失败时**：提炼错误归因 + 修复步骤（负样本）
- 确保两种记忆样本都有沉淀

## 分层上下文压缩

### 四阶段逐级执行

```
摘要预览 → 占位替换 → 按需检索 → 超限兜底
```

- **摘要预览**：将大型上下文生成为结构化的摘要笔记
- **占位替换**：缓存友好型占位符压缩
- **按需检索**：需要时展开占位符恢复详情
- **超限兜底**：上下文超限时的最终处理手段

### 触发条件（OR 关系）

任一条件触发即执行压缩：
- **Agent 自省**：Agent 评估认为上下文需整理
- **轮数阈值**：达到预设对话轮数
- **Token 阈值**：上下文使用率超过阈值

### 设计目标

- 大工具结果外置化
- 缓存友好型占位压缩
- 结构化笔记摘要
- 保证长会话稳定性
- 提升 Prompt Cache 收益
- 降低 Token 成本

## 中心化多 Agent 协作

### 架构原则

- 主 Agent 统一规划、审批与质量控制
- 子 Agent 以 Tool Call 方式受控执行
- 避免引入复杂的 Agent 间协调与状态管理基础设施
- 支持 Fork / Worktree 等协作模式

### 执行流程

```
主 Agent 制定计划
  ↓ (Tool Call 调度)
子 Agent 接收到目标
  ↓ (自规划执行)
子 Agent 调用 Tool → 执行步骤
  ↓ (返回结果)
主 Agent 评估、审批、下一步决策
```
