Memory System：三种记忆类型

Lesson 005 · 阅读时间 7 分钟

这是框架中最具「自进化」色彩的部分——Agent 不再每次从零开始，而是能把执行中获得的经验沉淀下来，跨会话复用。

三种记忆类型

📄

FactualMemory

静态事实知识：API 参数、配置值、环境信息。

来源：从执行输出中用正则提取 key=value 模式。

key="api_endpoint" → content="api_endpoint=staging.api.com"
📋

ProceduralMemory

程序性知识：How-to 步骤、成功模式。

来源：成功执行的输出步骤序列。

steps=["check: reviewed syntax", "report: generated output"]
💡

EpisodicMemory

情景记忆：失败事件 + 上下文环境。

来源：失败执行的错误信息 + 环境状态。

context={"error": "connection refused", "completed_steps": 1}

生命周期

执行 → 反思 → 提炼 → 分类存储 → 索引更新 → 按需复用

其中「反思」和「提炼」由 ReflectionEngine 负责（Lesson 006），「分类存储」「索引更新」「按需复用」由 StorageBackend 负责。

数据类定义

@dataclass
class MemoryRecord:
    key: str              # 唯一标识，如 "proc_deploy_a1b2c3"
    content: str          # 记忆的文本内容
    tags: list[str]       # 检索标签
    type: str             # "factual" / "procedural" / "episodic"

@dataclass
class FactualMemory(MemoryRecord):
    source: str = ""      # 知识来源，如 "execution:deployment"

@dataclass
class ProceduralMemory(MemoryRecord):
    domain: str = ""      # 所属领域，如 "deployment"
    steps: list[str] = field(default_factory=list)  # 步骤列表

@dataclass
class EpisodicMemory(MemoryRecord):
    context: dict = field(default_factory=dict)  # 上下文环境
    outcome: str = ""     # "success" / "failure"

存储层：可插拔设计

class StorageBackend(ABC):
    """任何存储后端只需实现这三个方法"""
    @abstractmethod
    def store(self, record: MemoryRecord) -> None: ...
    @abstractmethod
    def query(self, memory_type, tags, limit=100) -> list[MemoryRecord]: ...
    @abstractmethod
    def cross_reference(self, key) -> list[str]: ...

内置的 SQLiteStorageBackend：

  • 使用 SQLite 表存储，列：id, key, type, content, tags(JSON), extra_data(JSON)
  • 维护内存中的 tag → set[keys] 索引，cross_reference 是 O(1) 而非 O(n)
  • 跨会话持久化——数据写入了磁盘文件，下次启动仍在

跨会话复用的含义

在 SQLite 模式下：

# Session 1：执行 deploy，反思沉淀记忆
engine.reflect(exec_result, intent_class="deployment")

# 程序退出，下次启动...
# Session 2：查询记忆系统，发现上次的经验
storage = SQLiteStorageBackend("sigma.db")
past_memories = storage.query(memory_type="procedural", tags=["deployment"])
# → 返回上次存储的部署步骤经验

这就是"自进化"的含义——不是代码自动修改自己，而是每次执行的经验被自动提炼为可查询的记忆资产，新会话可以按需加载。

澄清：「自进化」不是说 Agent 自己改代码，而是：执行 → 反思 → 提炼 → 复用，这个闭环在每次执行后自动运行，不需要人工干预。经验越积越多，Agent 的表现自然「进化」。

要记住的一句话：三种记忆类型对应了人类学习中的「知道什么」（事实）、「知道怎么做」（程序）、「记得那次翻车」（情景）。一套统一的接口管理三种不同类型的经验。
