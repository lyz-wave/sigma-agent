ReflectionEngine：执行后反思

Lesson 006 · 阅读时间 6 分钟 · ← Memory System

Router 选 Skill → Executor 执行 → 然后呢？然后 ReflectionEngine 开始工作。它是 Skill Executor 和 Memory System 之间的桥梁。

一句话概括

每次 Skill 执行完成后（无论成功失败），ReflectionEngine 自动：

  • 成功：提取执行步骤 → 存为程序性记忆（怎么做成的）
  • 成功：提取 key=value 知识 → 存为事实记忆（学到了什么）
  • 失败：记录错误上下文 → 存为情景记忆（当时环境是什么）
  • 失败：生成修复步骤 → 存为程序性记忆（下次怎么避免）

而且可以一键关闭：ReflectionEngine(storage, enabled=False)。

双路径策略

✅ 成功路径

产出 ProceduralMemory + FactualMemory

输入：ExecutionResult(success=True,
  outputs=["check: reviewed syntax",
           "report: generated report"])

输出：
→ ProceduralMemory
  key="proc_code_review_a1b2c3"
  steps=["check: reviewed syntax",
         "report: generated report"]
  tags=["code_review", "review"]

→ FactualMemory
  key="fact_branch_fe1234"   ← 从输出中提取
  content="branch=main"
  source="execution:code_review"
  tags=["code_review"]

❌ 失败路径

产出 EpisodicMemory + ProceduralMemory

输入：ExecutionResult(success=False,
  outputs=["config: loaded"],
  error="Connection refused")

输出：
→ EpisodicMemory
  key="epi_deploy_b2c3d4"
  context={"error": "Connection refused",
           "completed_steps": 1}
  outcome="failure"
  tags=["deployment", "failure"]

→ ProceduralMemory (修复指南)
  key="proc_deploy_fix_c3d4e5"
  steps=[
    "1. Review the error details",
    "2. Check the failing step config",
    "3. Fix and retry (deployment)",
  ]
  tags=["deployment", "fix"]

代码流程

def reflect(self, result, intent_class, query_context):
    if not self._enabled:
        return                  # ← 可以一键关闭

    if result.success:
        self._reflect_success(...)
    else:
        self._reflect_failure(...)

两个分支各自做三件事：

  1. 解析执行结果 — 从 outputs 中提取结构化信息
  2. 构造记忆对象 — 选择合适的记忆类型和字段
  3. 写入存储 — self.storage.store(mem)

FactualMemory 提取细节

成功路径中，_extract_factual_memory() 扫描每个输出：

输出："generated key=abc123 endpoint=api.example.com"
                         ↓ 正则匹配
正则：([a-zA-Z_]\w*)\s*=\s*([a-zA-Z0-9_/.@-]+)
                         ↓ 提取结果
facts = [("key", "abc123"), ("endpoint", "api.example.com")]
                         ↓ 逐条存储
→ FactualMemory(key="fact_key_f4ce03", content="key=abc123")
→ FactualMemory(key="fact_endpoint_f4ce03", content="endpoint=api.example.com")

这样，Agent 执行一次后就知道 endpoint=api.example.com 了——下次需要时可以直接从记忆系统查询，而不需要再执行一次。

可配置性

当前硬编码的标签推断逻辑：

if "deploy" in query_context.lower(): tags.append("deployment")
if "test" in query_context.lower(): tags.append("testing")
if "review" in query_context.lower(): tags.append("review")

这在实际使用时应该替换为更灵活的映射，但核心模式不变：成功→程序性+事实记忆，失败→情景+修复步骤。

要记住的一句话：ReflectionEngine 让"吃一堑长一智"在 Agent 身上成为现实——不是通过修改代码，而是通过自动提炼经验存入记忆系统。成功和失败都被视为学习机会，只是学习的内容不同。
