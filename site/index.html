四大技术支柱

从路由到记忆，从上下文到协作 — 四个维度构建完整的 Agent 执行闭环

Skill 分层路由
原子 Tool → 高层 Skill → Skill 目录三层结构。两阶段召回（意图+标签）与精排（打分函数），解决 Tool 数量增长后的检索噪声与功能重叠。
Stage 1 Recall Stage 2 Scoring No LLM at Rank
02
🧠
自进化记忆沉淀
执行 → 反思 → 提炼 → 分类存储 → 索引更新 → 按需复用。成功经验与失败归因双路径沉淀，三种记忆类型（事实/程序性/情景）自动分类。
Factual Procedural Episodic
03
📦
分层上下文压缩
摘要预览 → 占位替换 → 按需检索 → 超限兜底，四阶段逐级执行。三种 OR 触发条件（自省/轮数/Token）。缓存友好，降低 60%+ Token 成本。
Summary Preview Placeholder Prompt Cache
04
🤝
中心化多 Agent 协作
主 Agent 统一规划审批，子 Agent 以 Tool Call 方式受控执行。Fork（进程内并行）与 Worktree（隔离工作区）双模式，避免复杂协调基础设施。

Skill Commands

Slash Commands

注册 Skill → 路由匹配 → 执行编排 → 反思沉淀，一条命令完成完整闭环

sigma-agent — interactive
$ sigma.route --intent code_review --tags python,quality
→ matched: code-review (score 0.87)

$ sigma.execute code-review
→ tool:check → tool:report → done
→ memory: procedural stored (key=proc_code_review_a1b2c3)

$ sigma.plan "build, test, and deploy"
→ 3 subtasks: build → test → deploy
→ mode: sequential (worktree=on)
→ all 3/3 subtasks passed ✓
