# Learning Record 003 — Stage 1 召回

**日期**：2026-07-14

**学到什么**：
- Stage 1 是"快滤"不是"精排"——目标是快速排除不相关 Skill，而非精确排序
- 两条召回路径：意图路径（精确匹配，精而少）vs 标签路径（重叠匹配，宽而广）
- 意图分类和标签提取当前是 Router 的外部输入，框架未内置实现
- 分类方式高度依赖场景，框架选择让使用者自行接入

**关键洞察**：
- Stage 1 是唯一适合调 LLM 的路由环节——一次过滤调用可以排除大量候选，后续都不再需要 LLM
- 标签的设计目的是捕捉"意图之外的上下文"——同样是 deploy，deploy to k8s 和 deploy to serverless 的标签截然不同
- 两条路径的设计是一个"先确定后兜底"的防御性策略
