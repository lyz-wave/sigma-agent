# Learning Record 002 — Router 精排打分函数

**日期**：2026-07-14

**学到什么**：
- 打分函数是三要素加权线性组合：意图匹配(0.5) + 标签重叠(0.3) + 示例相似度(0.2)
- "不用 LLM 做精排"是整个项目最重要的设计哲学——路由本身的成本不能抵消它的收益
- 权重可配置，适应不同的使用场景
- 有测试专门验证精排阶段无 LLM 调用

**关键洞察**：
- 三个维度的权重本身就是设计决策：意图主导(0.5)意味着"先看对不对，再看好不好"
- 平局时有发生——这是故意不解决，留给 Orchestrator 做最终判断
- 示例相似度用 query_text 参数传递真实用户输入（修复前用了 intent_class，实际为空操作）

**后续问题**：
- 是否需要引入语义相似度（embedding）替代关键词重叠？
- 实时场景下是否需要缓存打分结果？
